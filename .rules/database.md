# 🗄️ Database & Prisma Rules

## 1. Naming Conventions
- **Case Style**: Все таблицы и поля ОБЯЗАНЫ использовать `snake_case` (например, `user_profiles`, `transaction_id`).
- **Relations**: Используй понятные и описательные названия для связей (relations) в Prisma, избегая дефолтных имен.

## 2. Security & Safety
- **Row Level Security (RLS)**: Всегда проверяй и настраивай политики RLS при описании схем для Supabase. Пользователи должны иметь доступ только к своим данным (`auth.uid() = user_id`).
- **Migrations**: Прямые правки в БД запрещены. Все изменения структуры вносятся ТОЛЬКО через обновление `schema.prisma` и создание миграций.

## 3. Structure & Types
- **Enums**: Для полей с фиксированным набором значений (статусы, роли, категории) обязательно используй `enum`.
- **Metadata**: Для гибких данных используй тип `jsonb` с дефолтным значением `{}`.
- **Timestamps**: Каждая таблица должна иметь поля `created_at` и `updated_at` с автоматическим заполнением.
