# ADR-003: Интеграция стратегии UI Pro Max & Analytic Noir

## 1. Контекст (The Problem)
Стандартные интерфейсы, генерируемые ИИ, часто лишены визуальной целостности, имеют хаотичные отступы и не используют современные библиотеки компонентов. Для премиальных проектов Виктора (таких как GuitarOS) требуется уровень High-Fidelity, который сложно поддерживать вручную без строгих архитектурных рамок.

## 2. Предложенное решение (The Decision)
Принято решение внедрить трехуровневую стратегию проектирования интерфейсов:
1.  **Архитектурный скелет**: Использование навыка `uipro` (на базе `ui-ux-pro-max-skill`) для создания структуры экранов.
2.  **Атомарные компоненты**: Использование MCP-сервера `magic-ui` и библиотеки `21st.dev` для внедрения проверенных UI-паттернов.
3.  **Визуальный код**: Стиль "Analytic Noir" (Pure Black `#000000`, сетка 8pt, шрифт Geist, пружинные анимации Framer Motion).

## 3. Последствия (Consequences)

### ✅ Плюсы
- **Скорость**: Агент не тратит время на написание CSS с нуля, используя готовые шаблоны.
- **Качество**: Интерфейс соответствует стандартам индустрии "Pro Max".
- **Автоматизация**: Визуальная целостность проверяется через Playwright.

### ❌ Минусы / Риски
- **Зависимость**: Проект привязан к внешним библиотекам компонентов (`shadcn`, `21st.dev`).
- **Контекст**: Требуется постоянная поддержка актуальности `MAGIC_API_KEY` для работы MCP.
